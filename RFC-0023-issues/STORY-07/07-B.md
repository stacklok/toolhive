# Update MCPServer Controller for Config Ref Resolution

**Parent**: [STORY-07](STORY-07.md) — Workload CRD Config Reference Updates

## Description

Update the MCPServer controller to resolve config references to MCPOIDCConfig and MCPTelemetryConfig, merge per-server overrides, and implement fail-closed behavior.

## Implementation Notes

**File**: `cmd/thv-operator/controllers/mcpserver_controller.go`

### Config ref resolution

Add a resolution step early in the reconcile loop:

1. **OIDC resolution**:
   - If `spec.oidcConfig.ref` is set, fetch the referenced `MCPOIDCConfig`
   - If not found → set `Failed` phase, condition `OIDCConfigNotFound`, return (fail closed)
   - If found but not `Ready` → set `Failed` phase, condition `OIDCConfigNotReady`, return (fail closed)
   - Merge per-server overrides: `audience` (default to `MCPServer/{namespace}/{name}` if empty), `scopes`
   - If `spec.oidcConfig.inline` is set, use inline config directly (no resolution needed)

2. **Telemetry resolution**:
   - If `spec.telemetryConfig.ref` is set, fetch the referenced `MCPTelemetryConfig`
   - If not found → set `Failed` phase, condition `TelemetryConfigNotFound`, return (fail closed)
   - If found but not `Ready` → set `Failed` phase, condition `TelemetryConfigNotReady`, return (fail closed)
   - Merge per-server `serviceName`
   - If `spec.telemetryConfig.inline` is set, use inline config directly

### Config hash tracking

Add annotations/status fields for change detection:
- `toolhive.stacklok.dev/oidcconfig-hash` — set by MCPOIDCConfig controller on cascade
- `toolhive.stacklok.dev/telemetryconfig-hash` — set by MCPTelemetryConfig controller on cascade
- The MCPServer controller should watch for annotation changes and re-reconcile

### Watch setup

Ensure the MCPServer controller watches MCPOIDCConfig and MCPTelemetryConfig resources via `EnqueueRequestsFromMapFunc` (mapping config changes to MCPServer reconciliation). This may already be handled by the config controllers writing annotations, but explicit watches ensure no edge cases.

## Testing

**Unit tests**:
- Happy path: ref to Ready MCPOIDCConfig → resolved config with merged audience/scopes
- Happy path: ref to Ready MCPTelemetryConfig → resolved config with merged serviceName
- Inline fallback: inline config used directly, no resolution
- Fail-closed: ref to non-existent MCPOIDCConfig → Failed phase, `OIDCConfigNotFound` condition
- Fail-closed: ref to non-Ready MCPOIDCConfig → Failed phase, `OIDCConfigNotReady` condition
- Same fail-closed tests for MCPTelemetryConfig
- Default audience: no audience specified → `MCPServer/{namespace}/{name}` generated
- Override merging: audience and scopes from ref site override shared config

**Integration tests** in `cmd/thv-operator/test-integration/`:

Cross-CRD reference resolution:
1. Create MCPOIDCConfig + MCPTelemetryConfig → both reach Ready
2. Create MCPServer referencing both → verify MCPServer reaches Ready
3. Verify config hashes propagated to MCPServer annotations
4. Update MCPOIDCConfig → verify MCPServer re-reconciles
5. Delete MCPOIDCConfig (blocked by finalizer) → delete MCPServer → delete MCPOIDCConfig succeeds

Fail-closed integration:
1. Create MCPServer referencing non-existent MCPOIDCConfig → verify Failed phase
2. Create the MCPOIDCConfig → verify MCPServer transitions to Ready
3. Delete MCPOIDCConfig (or make it non-Ready) → verify MCPServer transitions to Failed

## Acceptance Criteria

- [ ] OIDC config ref resolved with per-server audience/scopes merge
- [ ] Telemetry config ref resolved with per-server serviceName merge
- [ ] Fail closed on missing config ref (phase=Failed, descriptive condition)
- [ ] Fail closed on non-Ready config ref
- [ ] Default audience generated as `MCPServer/{namespace}/{name}` when not specified
- [ ] Inline config path still works (no resolution needed)
- [ ] Config hash annotations tracked for change detection
- [ ] Unit tests cover all resolution paths, fail-closed, default audience, and override merging
- [ ] Integration tests cover cross-CRD reference lifecycle and fail-closed with recovery
- [ ] All tests pass in CI (`task test`)
