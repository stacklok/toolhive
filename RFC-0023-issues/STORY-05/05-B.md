# Update MCPRegistry Controller for Conditions-Based Status

**Parent**: [STORY-05](STORY-05.md) — MCPRegistry Status Consolidation

## Description

Update the MCPRegistry controller to set `SyncReady` and `APIReady` conditions instead of sub-phase enums. Ensure `Status.Phase` is only set via `DeriveOverallPhase()`.

## Implementation Notes

**Controller file**: `cmd/thv-operator/controllers/mcpregistry_controller.go` (and related files)

**Changes**:

1. **Replace phase-setting code**: Wherever the controller currently sets `status.SyncStatus.Phase` or `status.APIStatus.Phase`, replace with `meta.SetStatusCondition()` calls:
   ```go
   meta.SetStatusCondition(&registry.Status.Conditions, metav1.Condition{
       Type:               "SyncReady",
       Status:             metav1.ConditionTrue,
       Reason:             "SyncCompleted",
       Message:            "Registry sync completed successfully",
       ObservedGeneration: registry.Generation,
   })
   ```

2. **Derive phase from conditions**: After setting conditions, always call `DeriveOverallPhase()` to compute the top-level `Status.Phase`. Never set `Status.Phase` directly.

3. **Condition transitions**: Map existing phase transitions to condition updates:
   - Sync started → `SyncReady=False`, reason `Syncing`
   - Sync completed → `SyncReady=True`, reason `SyncCompleted`
   - Sync failed → `SyncReady=False`, reason `SyncFailed`, message with error
   - API deploying → `APIReady=False`, reason `Deploying`
   - API ready → `APIReady=True`, reason `DeploymentAvailable`
   - API unhealthy → `APIReady=False`, reason `Unhealthy`

4. **Status subresource update**: Continue using `Status().Update()` for status changes.

## Testing

- Sync lifecycle: trigger sync → verify `SyncReady=False/Syncing` → complete → verify `SyncReady=True/SyncCompleted`
- Sync failure: trigger sync → fail → verify `SyncReady=False/SyncFailed` with error message
- API lifecycle: deploy → verify `APIReady=False/Deploying` → ready → verify `APIReady=True/DeploymentAvailable`
- API unhealthy: deployment becomes unhealthy → verify `APIReady=False/Unhealthy`
- Phase derivation: after each condition change, verify `Status.Phase` matches expected value from `DeriveOverallPhase()`

## Acceptance Criteria

- [ ] Controller sets `SyncReady` and `APIReady` conditions with appropriate reasons and messages
- [ ] `Status.Phase` only set via `DeriveOverallPhase()`, never directly
- [ ] All sync lifecycle states map to correct condition transitions
- [ ] All API lifecycle states map to correct condition transitions
- [ ] `meta.SetStatusCondition()` used for proper condition management
- [ ] Unit tests cover all controller condition transitions for sync and API lifecycles
- [ ] Tests pass in CI (`task test`)
