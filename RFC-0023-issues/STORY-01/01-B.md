# Add CEL Validation Rule to `AuthzConfigRef`

**Parent**: [STORY-01](STORY-01.md) — CEL Validation on Existing Union Types

## Description

Add a `+kubebuilder:validation:XValidation` marker to the `AuthzConfigRef` struct in `mcpserver_types.go` that enforces exactly one of `configMapRef` or `inline` is set.

## Implementation Notes

**Target struct**: `AuthzConfigRef` in `cmd/thv-operator/api/v1alpha1/mcpserver_types.go` (lines 594-610).

**CEL rule**:
```go
// +kubebuilder:validation:XValidation:rule="(has(self.configMapRef) ? 1 : 0) + (has(self.inline) ? 1 : 0) == 1",message="exactly one of configMapRef or inline must be set"
```

**Key files**:
- `cmd/thv-operator/api/v1alpha1/mcpserver_types.go` — struct definition
- Run `task gen` after changes to regenerate manifests

## Testing

Integration tests in `cmd/thv-operator/test-integration/` covering all `AuthzConfigRef` permutations:

- Valid: only `inline` set → accepted
- Valid: only `configMapRef` set → accepted
- Invalid: both set → rejected
- Invalid: neither set → rejected

Tests should apply manifests against the envtest API server and assert the expected validation error messages.
