# Add CEL Validation Rule to `AuthzConfigRef`

**Parent**: [STORY-01](STORY-01.md) — CEL Validation on Existing Union Types

## Description

Add `+kubebuilder:validation:XValidation` markers to the `AuthzConfigRef` struct in `mcpserver_types.go` that enforce the populated field matches the `type` discriminator. The existing JSON field names are `configMap` and `inline` (not `configMapRef`).

## Implementation Notes

**Target struct**: `AuthzConfigRef` in `cmd/thv-operator/api/v1alpha1/mcpserver_types.go` (lines 594-610).

**CEL rules** (per-field, matching `MCPExternalAuthConfig` pattern):
```go
// +kubebuilder:validation:XValidation:rule="self.type == 'configMap' ? has(self.configMap) : !has(self.configMap)",message="configMap must be set when type is 'configMap' and unset otherwise"
// +kubebuilder:validation:XValidation:rule="self.type == 'inline' ? has(self.inline) : !has(self.inline)",message="inline must be set when type is 'inline' and unset otherwise"
```

> **Note**: `AuthzConfigRef` uses JSON field name `configMap` (not `configMapRef`), and has a `type` discriminator with enum `configMap;inline`.

**Key files**:
- `cmd/thv-operator/api/v1alpha1/mcpserver_types.go` — struct definition
- Run `task gen` after changes to regenerate manifests

## Testing
