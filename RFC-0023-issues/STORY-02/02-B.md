# Define `OIDCConfigWithRef` Struct for Workload References

**Parent**: [STORY-02](STORY-02.md) — MCPOIDCConfig CRD

## Description

Create a reference struct for use in workload CRDs (MCPServer, MCPRemoteEndpoint, VirtualMCPServer) that allows either referencing a shared `MCPOIDCConfig` or specifying inline OIDC configuration, with per-server override fields.

## Implementation Notes

**Struct definition** (in a shared types file or alongside MCPOIDCConfig types):

```go
// OIDCConfigWithRef allows referencing a shared MCPOIDCConfig or specifying inline config.
// +kubebuilder:validation:XValidation:rule="(has(self.ref) ? 1 : 0) + (has(self.inline) ? 1 : 0) == 1",message="exactly one of ref or inline must be set"
type OIDCConfigWithRef struct {
    // Ref references a same-namespace MCPOIDCConfig resource.
    Ref *LocalObjectReference `json:"ref,omitempty"`

    // Inline specifies OIDC configuration directly (for simple single-server deployments).
    Inline *InlineOIDCProviderConfig `json:"inline,omitempty"`

    // Audience for this specific server. MUST be unique per server to prevent token replay.
    // Defaults to {kind}/{namespace}/{name} if not specified.
    // +optional
    Audience string `json:"audience,omitempty"`

    // Scopes requested for this specific server.
    // +optional
    Scopes []string `json:"scopes,omitempty"`
}
```

**Key design decisions**:
- `audience` defaults to `{kind}/{namespace}/{name}` (e.g., `MCPServer/default/github-mcp`) — computed by the workload controller, not stored in the CRD
- `scopes` may vary per server depending on required access
- CEL rule enforces `ref` and `inline` are mutually exclusive
- Same-namespace references only (no `namespace` field in `Ref`)

## Acceptance Criteria

- [ ] `OIDCConfigWithRef` struct defined with `ref`, `inline`, `audience`, `scopes`
- [ ] CEL rule enforces mutual exclusivity of `ref` and `inline`
- [ ] `audience` and `scopes` are optional per-server overrides
- [ ] No `namespace` field in `Ref` (same-namespace only)
- [ ] `task gen` succeeds
