# Helm Chart — MCPOIDCConfig CRD Template

**Parent**: [STORY-02](STORY-02.md) — MCPOIDCConfig CRD

## Description

Add the generated MCPOIDCConfig CRD manifest to the `operator-crds` Helm chart so that it is installed alongside other ToolHive CRDs.

## Implementation Notes

**Helm chart location**: The `operator-crds` Helm chart is separate from the operator chart — CRDs are installed before the controller for upgrade safety.

**Steps**:
1. After `task gen` produces the CRD manifest, copy it to `deploy/charts/operator-crds/files/crds/`
2. Run the `crd-helm-wrapper` tool to generate the wrapped Helm template in `deploy/charts/operator-crds/templates/`. Update the `crdFeatureFlags` map in `crd-helm-wrapper/main.go` to assign the new CRD to the appropriate feature flag group.
3. Verify `helm template` renders the CRD correctly
4. Verify `helm install` installs the CRD on a test cluster

**RBAC considerations**: Add example RBAC manifests or values for:
- Platform team: `get`, `list`, `watch`, `create`, `update`, `patch`, `delete` on `mcpoidcconfigs`
- Application team: `get`, `list`, `watch` on `mcpoidcconfigs` (read-only)

## Acceptance Criteria

- [ ] MCPOIDCConfig CRD manifest included in `operator-crds` Helm chart
- [ ] `helm template` renders the CRD correctly
- [ ] RBAC examples included for platform team / app team separation
