# Remove Deprecated Fields from MCPServer Types

**Parent**: [STORY-04](STORY-04.md) — Remove Deprecated Fields

## Description

Remove all deprecated fields from `MCPServerSpec` and `InlineOIDCConfig`, and simplify the helper methods that previously provided fallback logic.

## Implementation Notes

**File**: `cmd/thv-operator/api/v1alpha1/mcpserver_types.go`

**Fields to remove**:
1. `Port` (line ~96) — `proxyPort` is the canonical field
2. `TargetPort` (line ~103) — `mcpPort` is the canonical field
3. `ToolsFilter` (line ~175) — `toolConfigRef` is the canonical field
4. `InlineOIDCConfig.ClientSecret` (line ~541) — `clientSecretRef` is the canonical field
5. `InlineOIDCConfig.ThvCABundlePath` (line ~553) — `caBundleRef` is the canonical field

**Helper methods to simplify**:
- `GetProxyPort()` — remove the `if s.Port != 0 { return s.Port }` fallback, return `s.ProxyPort` directly
- `GetMcpPort()` — remove the `if s.TargetPort != 0 { return s.TargetPort }` fallback, return `s.McpPort` directly
- Any similar fallback helpers for `ToolsFilter`, `ClientSecret`, `ThvCABundlePath`
