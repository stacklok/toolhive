# Add Printer Columns to New Config CRDs

**Parent**: [STORY-06](STORY-06.md) — Printer Columns

## Description

Add `+kubebuilder:printcolumn` markers to `MCPOIDCConfig` and `MCPTelemetryConfig` so that `kubectl get` output is informative.

## Implementation Notes

**MCPOIDCConfig columns**:
```go
// +kubebuilder:printcolumn:name="Ready",type="string",JSONPath=`.status.conditions[?(@.type=="Ready")].status`
// +kubebuilder:printcolumn:name="References",type="integer",JSONPath=`.status.referencingServers`,description="Number of referencing workloads"
// +kubebuilder:printcolumn:name="Age",type="date",JSONPath=`.metadata.creationTimestamp`
```

Note: JSONPath for array length may need a custom approach. If direct JSONPath can't express `len(referencingServers)`, add a `referenceCount` integer field to status instead.

**MCPTelemetryConfig columns**:
```go
// +kubebuilder:printcolumn:name="Endpoint",type="string",JSONPath=`.spec.endpoint`
// +kubebuilder:printcolumn:name="Ready",type="string",JSONPath=`.status.conditions[?(@.type=="Ready")].status`
// +kubebuilder:printcolumn:name="References",type="integer",JSONPath=`.status.referencingServers`,description="Number of referencing workloads"
// +kubebuilder:printcolumn:name="Age",type="date",JSONPath=`.metadata.creationTimestamp`
```

**Existing pattern**: `MCPExternalAuthConfig` has `Type` and `Age` columns (`mcpexternalauthconfig_types.go:684-685`).

**Post-implementation**: Run `task gen`.

## Acceptance Criteria

- [ ] MCPOIDCConfig shows Ready, reference count, and Age in `kubectl get`
- [ ] MCPTelemetryConfig shows Endpoint, Ready, reference count, and Age in `kubectl get`
- [ ] JSONPath expressions resolve correctly
- [ ] `task gen` succeeds
