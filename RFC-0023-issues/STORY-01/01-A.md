# Add CEL Validation Rule to `OIDCConfigRef`

**Parent**: [STORY-01](STORY-01.md) — CEL Validation on Existing Union Types

## Description

Add `+kubebuilder:validation:XValidation` markers to the `OIDCConfigRef` struct in `mcpserver_types.go` that enforce the populated field matches the `type` discriminator. The existing JSON field names are `kubernetes`, `configMap`, `inline` (not `kubernetesServiceAccount` or `configMapRef`).

## Implementation Notes

**Pattern to follow**: `MCPExternalAuthConfig` CEL rules at `mcpexternalauthconfig_types.go:44-51`:
```go
// +kubebuilder:validation:XValidation:rule="self.type == 'tokenExchange' ? has(self.tokenExchange) : !has(self.tokenExchange)"
```

**Target struct**: `OIDCConfigRef` in `cmd/thv-operator/api/v1alpha1/mcpserver_types.go` (lines 422-448).

**CEL rules** (per-field, matching `MCPExternalAuthConfig` pattern):
```go
// +kubebuilder:validation:XValidation:rule="self.type == 'kubernetes' ? has(self.kubernetes) : !has(self.kubernetes)",message="kubernetes must be set when type is 'kubernetes' and unset otherwise"
// +kubebuilder:validation:XValidation:rule="self.type == 'configMap' ? has(self.configMap) : !has(self.configMap)",message="configMap must be set when type is 'configMap' and unset otherwise"
// +kubebuilder:validation:XValidation:rule="self.type == 'inline' ? has(self.inline) : !has(self.inline)",message="inline must be set when type is 'inline' and unset otherwise"
```

> **Note**: The existing `OIDCConfigRef` has a `type` discriminator field with enum `kubernetes;configMap;inline` and default `kubernetes`. The per-field CEL pattern (matching `type` to populated field) follows the `MCPExternalAuthConfig` precedent and is compatible with existing manifests that set `type` without an explicit sub-field (since `KubernetesOIDCConfig` fields are all optional, `has(self.kubernetes)` can be true with an empty object).

**Key files**:

- Invalid: `type: kubernetes` but `configMap` field set → rejected (type/field mismatch)
- Invalid: `type: inline` but `kubernetes` field set → rejected
