# Update Controllers and Tests for Deprecated Field Removal

**Parent**: [STORY-04](STORY-04.md) — Remove Deprecated Fields

## Description

Update all controller code and tests that reference the removed deprecated fields to use only the canonical field names.

## Implementation Notes

**Search for references**: Grep the codebase for the removed field names to find all usage sites:
- `\.Port` (in context of MCPServer/MCPRemoteProxy spec access)
- `\.TargetPort`
- `\.ToolsFilter`
- `\.ClientSecret` (in InlineOIDCConfig context)
- `\.ThvCABundlePath`
- `GetProxyPort` / `GetMcpPort` callers (these still exist but are simplified)

**Controller files likely affected**:
- `cmd/thv-operator/controllers/mcpserver_controller.go`
- `cmd/thv-operator/controllers/mcpremoteproxy_controller.go`
- Any conversion or validation code

**Test files to update**:
- All test files that create MCPServer or MCPRemoteProxy fixtures using deprecated fields
- Update to use `ProxyPort`, `McpPort`, `ToolConfigRef`, `ClientSecretRef`, `CABundleRef`

**Migration notes**: Create a brief mapping table in the PR description documenting the exact field changes for users upgrading from v1alpha1.

## Acceptance Criteria

- [ ] No remaining references to removed fields in controller code
- [ ] All unit tests updated to use canonical field names
- [ ] All tests pass (`task test`)
- [ ] `task lint` passes
