# CRD Wrapping for Conditional Installation

This directory contains scripts for wrapping ToolHive Operator CRDs with Helm template conditionals, allowing them to be optionally installed as part of the Helm chart.

## Overview

The CRD wrapping process takes the raw CRD YAML files generated by controller-gen and wraps them with Helm template conditionals, enabling users to control whether CRDs are installed during a Helm deployment.

## Architecture

```
deploy/
├── operator-crds/                    # Source: Raw CRDs generated by controller-gen
│   ├── toolhive.stacklok.dev_*.yaml
│   └── ...
├── charts/
│   └── operator/
│       ├── scripts/
│       │   ├── wrap-crds.sh         # Main script for wrapping CRDs
│       │   ├── crd-header.tpl       # Header template for Helm conditionals
│       │   └── crd-footer.tpl       # Footer template to close conditionals
│       └── templates/
│           └── definitions/          # Target: Wrapped CRDs with Helm conditionals
│               ├── toolhive.stacklok.dev_*.yaml
│               └── ...
```

## How It Works

1. **CRD Generation**: The `task operator-manifests` command runs controller-gen to generate raw CRD YAML files into `deploy/operator-crds/`

2. **CRD Wrapping**: The `wrap-crds.sh` script:
   - Reads each CRD file from `deploy/operator-crds/`
   - Wraps it with Helm template conditionals (`{{- if .Values.crds.install }}`)
   - Outputs the wrapped version to `deploy/charts/operator/templates/definitions/`

3. **Helm Installation**: Users can control CRD installation via the `crds.install` value:
   - `--set crds.install=true`: CRDs will be installed
   - `--set crds.install=false`: CRDs will be skipped (default)

## Usage

### Generating and Wrapping CRDs

Run the operator-manifests task from the operator directory:

```bash
cd cmd/thv-operator
task operator-manifests
```

This will:
1. Generate raw CRDs to `deploy/operator-crds/`
2. Wrap them with Helm templates in `deploy/charts/operator/templates/definitions/`
3. Generate RBAC manifests

### Installing the Helm Chart

#### With CRDs
```bash
helm install toolhive-operator deploy/charts/operator \
  --namespace toolhive-system \
  --create-namespace \
  --set crds.install=true
```

#### Without CRDs (default)
```bash
helm install toolhive-operator deploy/charts/operator \
  --namespace toolhive-system \
  --create-namespace
```

Or explicitly:
```bash
helm install toolhive-operator deploy/charts/operator \
  --namespace toolhive-system \
  --create-namespace \
  --set crds.install=false
```

### Skipping Specific CRDs

You can skip specific CRDs during installation using the `crds.skip` value:

```bash
helm install toolhive-operator deploy/charts/operator \
  --set crds.install=true \
  --set-string crds.skip='{mcpservers.toolhive.stacklok.dev,mcpgroups.toolhive.stacklok.dev}'
```

## Configuration

The following values control CRD installation in `values.yaml`:

```yaml
crds:
  # Whether to install CRDs as part of the Helm release
  install: false
  # List of CRD names to skip during installation
  skip: []
```

## Development Notes

### Adding New CRDs

When adding new CRDs:
1. Define them in the appropriate `api/v1alpha1/*.go` files
2. Run `task operator-generate` to generate DeepCopy methods
3. Run `task operator-manifests` to generate and wrap the CRDs
4. The new CRDs will automatically be included in the Helm chart

### Modifying the Wrapping Process

To modify how CRDs are wrapped:
1. Edit the template files:
   - `crd-header.tpl` - Controls the Helm conditional header
   - `crd-footer.tpl` - Closes the Helm conditional block
2. Edit `wrap-crds.sh` to change the wrapping logic if needed
3. Run `task operator-manifests` to regenerate wrapped CRDs

The script uses the template files to wrap each CRD with proper Helm conditionals, ensuring clean separation between the wrapping logic and the template syntax.

### Testing

To verify the CRD wrapping is working:

```bash
# Test that CRDs are included when enabled
helm template test deploy/charts/operator --set crds.install=true | grep "kind: CustomResourceDefinition"

# Test that CRDs are excluded when disabled
helm template test deploy/charts/operator --set crds.install=false | grep "kind: CustomResourceDefinition"
# Should return nothing
```

## Design Decisions

1. **Individual Files**: Each CRD is wrapped in its own file for better organization and git diff visibility
2. **Conditional Installation**: CRDs are not installed by default to avoid conflicts in environments where CRDs are managed separately
3. **Skip List**: The skip functionality allows for partial CRD updates during upgrades
4. **Backward Compatibility**: The legacy `deploy/operator/files/crds` path is maintained temporarily for compatibility
