# Define `TelemetryConfigWithRef` Struct for Workload References

**Parent**: [STORY-03](STORY-03.md) — MCPTelemetryConfig CRD

## Description

Create a reference struct for use in workload CRDs that allows either referencing a shared `MCPTelemetryConfig` or specifying inline telemetry configuration, with a required per-server `serviceName` field.

## Implementation Notes

**Struct definition**:

```go
// TelemetryConfigWithRef allows referencing a shared MCPTelemetryConfig or specifying inline config.
// +kubebuilder:validation:XValidation:rule="(has(self.ref) ? 1 : 0) + (has(self.inline) ? 1 : 0) == 1",message="exactly one of ref or inline must be set"
type TelemetryConfigWithRef struct {
    // Ref references a same-namespace MCPTelemetryConfig resource.
    Ref *LocalObjectReference `json:"ref,omitempty"`

    // Inline specifies telemetry configuration directly.
    Inline *MCPTelemetryConfigSpec `json:"inline,omitempty"`

    // ServiceName for this specific server. MUST be unique per server for observability.
    // +kubebuilder:validation:Required
    ServiceName string `json:"serviceName"`
}
```

**Key design decisions**:
- `serviceName` is required and per-server — each server must have a unique service name for metrics/traces
- CEL rule enforces `ref` and `inline` are mutually exclusive
- Same-namespace references only (no `namespace` field in `Ref`)

## Acceptance Criteria

- [ ] `TelemetryConfigWithRef` struct defined with `ref`, `inline`, `serviceName`
- [ ] CEL rule enforces mutual exclusivity of `ref` and `inline`
- [ ] `serviceName` is required
- [ ] `task gen` succeeds
