# Refactor MCPRegistry Status Types

**Parent**: [STORY-05](STORY-05.md) — MCPRegistry Status Consolidation

## Description

Replace the three phase enums in MCPRegistry status with a conditions-based pattern. Remove `SyncPhase` and `APIPhase` enums, add standard Kubernetes conditions, and update `DeriveOverallPhase()` to compute from conditions.

## Implementation Notes

**File**: `cmd/thv-operator/api/v1alpha1/mcpregistry_types.go`

**Changes to `MCPRegistryStatus`** (lines 485-521):
1. Repurpose the existing `Conditions []metav1.Condition` field (already present at lines 516-520 with `+listType=map` and `+listMapKey=type`)
2. Reuse existing condition type constant `ConditionAPIReady` ("APIReady", line 657). Add `SyncReady` condition type (or repurpose existing `ConditionSyncSuccessful` at line 654).

> **Note**: `MCPRegistryStatus` already has `Conditions`, `ConditionAPIReady`, `ConditionSyncSuccessful`, `ConditionDataValid`, and `ConditionRegistryPodTemplateValid`. This sub-issue does NOT add `Conditions` — it changes how the controller populates them and removes the redundant `SyncPhase`/`APIPhase` enums.

**Changes to `SyncStatus`** (lines 523-555):
- Remove `Phase SyncPhase` field
- Retain: `LastAttempt`, `AttemptCount`, `LastSyncTime`, `LastSyncHash`, `ServerCount`

**Changes to `APIStatus`** (lines 557-574):
- Remove `Phase APIPhase` field
- Retain: `Endpoint` (line 569), `ReadySince` (line 573)
- Add: `Replicas` and `AvailableReplicas` as new fields (these do NOT currently exist — sourced from Deployment status)
- Remove: `Message` (captured in condition message instead)

**Remove enums**:
- `SyncPhase` type and constants (Syncing/Complete/Failed)
- `APIPhase` type and constants (NotStarted/Deploying/Ready/Unhealthy/Error)

**Update `DeriveOverallPhase()`** (line ~713):
- Compute `MCPRegistryPhase` from conditions:
  - Both `SyncReady=True` and `APIReady=True` → `Ready`
  - Any condition `False` with `reason=...Failed` → `Failed`
  - `SyncReady=False` with `reason=Syncing` → `Syncing`
  - Default → `Pending`

**Target status shape** (v1alpha2):
```yaml
status:
  phase: Ready
  conditions:
    - type: SyncReady
      status: "True"
      reason: SyncCompleted
    - type: APIReady
      status: "True"
      reason: DeploymentAvailable
  sync:
    lastSyncTime: "..."
    lastSyncHash: "sha256:..."
    serverCount: 42
  api:
    endpoint: "https://..."
    replicas: 2
    availableReplicas: 2
```

**Printer columns**: Update to show condition status instead of sub-phases. Existing columns at `mcpregistry_types.go:674-679`.

**Post-implementation**: Run `task gen`.
