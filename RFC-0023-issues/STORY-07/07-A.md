# Update MCPServer Types with Config Refs

**Parent**: [STORY-07](STORY-07.md) — Workload CRD Config Reference Updates

## Description

Replace inline OIDC and telemetry configuration in `MCPServerSpec` with the new ref+inline union structs (`OIDCConfigWithRef` from 02-B, `TelemetryConfigWithRef` from 03-B). Keep inline as a supported path for simple single-server deployments.

## Implementation Notes

**File**: `cmd/thv-operator/api/v1alpha1/mcpserver_types.go`

**Changes to `MCPServerSpec`**:

Replace the current `OIDCConfig OIDCConfigRef` field with:
```go
// OIDCConfig specifies OIDC authentication configuration via reference or inline.
// +optional
OIDCConfig *OIDCConfigWithRef `json:"oidcConfig,omitempty"`
```

Replace the current inline telemetry config (if any `OpenTelemetryConfig` field exists) with:
```go
// TelemetryConfig specifies telemetry configuration via reference or inline.
// +optional
TelemetryConfig *TelemetryConfigWithRef `json:"telemetryConfig,omitempty"`
```

**Breaking changes**:
- The `oidcConfig` field structure changes from `OIDCConfigRef` (with `type` discriminator) to `OIDCConfigWithRef` (with `ref`/`inline` union)
- Existing manifests using `oidcConfig.type: inline` + `oidcConfig.inline: {...}` must be rewritten to `oidcConfig.inline: {...}` (no `type` field)
- `audience` and `scopes` move from inside `oidcConfig.inline` to `oidcConfig.audience` / `oidcConfig.scopes` (per-server overrides)

**Post-implementation**: Run `task gen`.

## Acceptance Criteria

- [ ] `MCPServerSpec.OIDCConfig` uses `OIDCConfigWithRef` type
- [ ] `MCPServerSpec.TelemetryConfig` uses `TelemetryConfigWithRef` type
- [ ] CEL rules enforce `ref` and `inline` mutual exclusivity
- [ ] Old `OIDCConfigRef` with `type` discriminator no longer used in MCPServer
- [ ] `task gen` succeeds and CRD manifests updated
