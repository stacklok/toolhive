# Update VirtualMCPServer Types and Controller

**Parent**: [STORY-07](STORY-07.md) â€” Workload CRD Config Reference Updates

## Description

Add OIDC config reference support to VirtualMCPServer's `IncomingAuth` section. VirtualMCPServer does not use MCPTelemetryConfig (telemetry is configured via its embedded `config.Config`).

## Implementation Notes

**File**: `cmd/thv-operator/api/v1alpha1/virtualmcpserver_types.go`

**Current state**: VirtualMCPServer's `IncomingAuth` (lines 86-107) reuses MCPServer's `OIDCConfigRef` and `AuthzConfigRef`.

**Change**: Replace the `OIDCConfigRef` usage in `IncomingAuth` with `OIDCConfigWithRef`:
```go
type IncomingAuth struct {
    // OIDCConfig for incoming request validation.
    // +optional
    OIDCConfig *OIDCConfigWithRef `json:"oidcConfig,omitempty"`

    // AuthzConfig for authorization policy evaluation.
    // +optional
    AuthzConfig *AuthzConfigRef `json:"authzConfig,omitempty"`
}
```

**Controller changes**:
- Resolve `incomingAuth.oidcConfig.ref` â†’ MCPOIDCConfig
- Fail closed if referenced config is missing or not Ready
- Default audience: `VirtualMCPServer/{namespace}/{name}`
- Reuse shared resolution helpers from 07-C

**Note**: VirtualMCPServer retains its embedded `config.Config` for vMCP-specific settings. Only OIDC auth is shared via the new config ref pattern.

**Config controller watch updates**: MCPOIDCConfig controller (02-C) must watch VirtualMCPServer for reference tracking.
